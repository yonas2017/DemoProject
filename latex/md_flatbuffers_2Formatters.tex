\chapter{Format Guidelines}
\hypertarget{md_flatbuffers_2Formatters}{}\label{md_flatbuffers_2Formatters}\index{Format Guidelines@{Format Guidelines}}
If you are interesting in contributing to the flatbuffers project, please take a second to read this document. Each language has it\textquotesingle{}s own set of rules, that are defined in their respective formatter/linter documents.

\doxysection*{Notes}


\begin{DoxyItemize}
\item Run the linter on the language you are working on before making a Pull Request.
\item DONT format/lint the generated code.
\end{DoxyItemize}

\doxysection*{Languages}

\doxysubsection*{C++}

C++ uses {\ttfamily clang-\/format} as it\textquotesingle{}s formatter. Run the following script {\ttfamily sh scripts/clang-\/format-\/git.\+sh}, and it should style the C++ code according to \href{https://google.github.io/styleguide/cppguide.html}{\texttt{ google style guide}}.

\doxysubsection*{Swift}

Swift uses swiftformat as it\textquotesingle{}s formatter. Take a look at \href{https://github.com/nicklockwood/SwiftFormat/blob/master/README.md\#how-do-i-install-it}{\texttt{ how to install here}}. Run the following command {\ttfamily swiftformat -\/-\/config swift.\+swiftformat .} in the root directory of the project

\doxysubsection*{Typescript}

Typescript uses eslint as it\textquotesingle{}s linter. Take a look at \href{https://eslint.org/docs/user-guide/getting-started}{\texttt{ how to install here}}. Run the following command {\ttfamily eslint ts/\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} -\/-\/ext .ts} in the root directory of the project
