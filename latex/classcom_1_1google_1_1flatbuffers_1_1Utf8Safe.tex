\doxysubsection{Detailed Description}
A set of low-\/level, high-\/performance static utility methods related to the UTF-\/8 character encoding. This class has no dependencies outside of the core JDK libraries.

There are several variants of UTF-\/8. The one implemented by this class is the restricted definition of UTF-\/8 introduced in Unicode 3.\+1, which mandates the rejection of "{}overlong"{} byte sequences as well as rejection of 3-\/byte surrogate codepoint byte sequences. Note that the UTF-\/8 decoder included in Oracle\textquotesingle{}s JDK has been modified to also reject "{}overlong"{} byte sequences, but (as of 2011) still accepts 3-\/byte surrogate codepoint byte sequences.

The byte sequences considered valid by this class are exactly those that can be roundtrip converted to Strings and back to bytes using the UTF-\/8 charset, without loss\+: 
\begin{DoxyPre} {\ttfamily \newlineArrays.equals(bytes, new String(bytes, Internal.UTF\_8).getBytes(Internal.UTF\_8))\newline}\end{DoxyPre}


See the Unicode Standard, Table 3-\/6. {\itshape UTF-\/8 Bit Distribution}, Table 3-\/7. {\itshape Well Formed UTF-\/8 Byte Sequences}.
