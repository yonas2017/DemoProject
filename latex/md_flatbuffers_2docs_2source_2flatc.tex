\chapter{Flat\+Buffers Compiler (\texorpdfstring{$<$}{<}tt\texorpdfstring{$>$}{>}flatc\texorpdfstring{$<$}{<}/tt\texorpdfstring{$>$}{>})}
\hypertarget{md_flatbuffers_2docs_2source_2flatc}{}\label{md_flatbuffers_2docs_2source_2flatc}\index{FlatBuffers Compiler ($<$tt$>$flatc$<$/tt$>$)@{FlatBuffers Compiler ($<$tt$>$flatc$<$/tt$>$)}}
The main compiler for Flat\+Buffers is called {\ttfamily flatc} and is used to convert schema definitions into generated code files for a variety of languages.

After \doxysectlink{md_flatbuffers_2docs_2source_2building}{building}{0} {\ttfamily flatc}, it is used as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{flatc\ [\ GENERATOR\_OPTIONS\ ]\ [\ -\/o\ PATH\ ]\ [-\/\ I\ PATH\ ]\ }
\DoxyCodeLine{\ \ FILES...\ }
\DoxyCodeLine{\ \ [\ -\/-\/\ BINARY\_FILES...\ ]}

\end{DoxyCode}



\begin{DoxyItemize}
\item The {\ttfamily GENERATOR\+\_\+\+OPTIONS} specify the language(s) to compile code for as well as various features to enable/disable.
\item The {\ttfamily -\/o PATH} specifies the path where the generated files are placed. It defaults to the current path if not specified.
\item The {\ttfamily -\/I PATH} specifies the paths where included schema files are located. It defaults to the current path if not specified.
\end{DoxyItemize}

\doxysection*{Input Files}

{\ttfamily FILES...} specifies one or more schema or data files to process. They are processed in the order provided.

\doxysubsection*{Schema Files}

For schema files, language specifiers indicate what languages to generate code for.


\begin{DoxyItemize}
\item {\ttfamily -\/-\/cpp}\+: C++
\item {\ttfamily -\/-\/java}\+: Java
\item {\ttfamily -\/-\/kotlin}\+: Kotlin
\item {\ttfamily -\/-\/csharp}\+: C\#
\item {\ttfamily -\/-\/go}\+: Golang
\item {\ttfamily -\/-\/python}\+: Python
\item {\ttfamily -\/-\/js}\+: Java\+Script
\item {\ttfamily -\/-\/ts}\+: Type\+Script
\item {\ttfamily -\/-\/php}\+: PHP
\item {\ttfamily -\/-\/dart}\+: Dart
\item {\ttfamily -\/-\/lua}\+: Lua
\item {\ttfamily -\/-\/lobster}\+: Lobster
\item {\ttfamily -\/-\/rust}\+: Rust
\item {\ttfamily -\/-\/swift}\+: Swift
\item {\ttfamily -\/-\/nim}\+: Nim
\end{DoxyItemize}

Additionally, adding\+:


\begin{DoxyItemize}
\item {\ttfamily -\/-\/grpc} Will generate RPC stub code for g\+RPC (not available in all languages)
\end{DoxyItemize}

\doxysubsection*{Data Files}

If {\ttfamily FILES...} contain data files, they can be exported to either a binary or JSON representation.


\begin{DoxyItemize}
\item {\ttfamily -\/-\/binary}, {\ttfamily -\/b}\+: Generate a binary file containing a serialized flatbuffer.
\item {\ttfamily -\/-\/json}, {\ttfamily -\/j}\+: Generate JSON file from a serialized flatbuffer.
\end{DoxyItemize}

Both options require the corresponding schema file to be included first in the list of {\ttfamily FILES...}.

=== "{}\+To Binary"{} \begin{DoxyVerb}To serialize the JSON data in `mydata.json` using the schema `myschema.fbs`:

```sh
flatc --binary myschema.fbs mydata.json
```

This will generate a `mydata_wire.bin` file containing the serialized 
flatbuffer data.
\end{DoxyVerb}
 === "{}\+To JSON"{} \begin{DoxyVerb}To convert the serialized binary flatbuffer `mydata.bin` using the schema 
`myschema.fbs` to JSON:

```sh
flatc --json myschema.fbs mydata.bin
```

This will generate a `mydata.json` file.
\end{DoxyVerb}
